`bv-sim` is the simulator behind the figures in [[./bias-variance-refresher|the bias–variance refresher]]. It exists for one reason: every claim in a learning-theory write-up should be reproducible from a few lines of code.

## What it does

The simulator fits polynomial-feature ridge regressors to a fixed cubic target $f^\star$ corrupted by Gaussian noise. For each setting of (degree $d$, sample size $n$, ridge strength $\lambda$, noise $\sigma$) it draws $K$ independent training sets, fits a model on each, and reports the empirical decomposition
$$
\hat R(x_0) \approx \widehat{\text{bias}^2}(x_0) + \widehat{\text{variance}}(x_0) + \sigma^2.
$$

<TwoColumns>
<div>

### Sweepable knobs

- model degree $d \in \{1, \dots, 12\}$
- sample size $n \in \{20, 50, 200, 1000\}$
- ridge $\lambda \in \{0, 10^{-3}, \dots, 10\}$
- noise $\sigma \in \{0, 0.1, 1\}$
- replicates $K$ (default 200)

</div>
<div>

### What you get back

- The three-term decomposition at each query $x_0$
- A U-shape plot of $\hat R$ vs. $d$
- A dartboard render of model fits on a single $x_0$
- Pickled fits for downstream analysis

</div>
</TwoColumns>

<Callout type="note" title="Static + reproducible">
`bv-sim` is a single Python file with no GPU dependencies. The figures on [[./bias-variance-refresher]] are committed alongside the script that produced them, so a fresh clone reproduces the post bit-for-bit.
</Callout>

This project also links back to [[research-papers]] because the simulator is intentionally aligned with the explainer it accompanies — running it is part of *reading* the post.
